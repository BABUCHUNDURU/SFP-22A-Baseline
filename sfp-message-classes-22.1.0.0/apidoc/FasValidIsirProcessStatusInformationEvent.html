Oracle SFP Messaging API 22.1.0.0

FasValidIsirProcessStatusInformationEvent (Outbound)

outboundisir
Message Class
FasValidIsirProcessStatusInformationEvent
Description

Message used to notify students of information about valid ISIRs per process for a given student.

See also FasValidIsirProcessStatusInformationRequest.

XML Schema

FasValidIsirProcessStatusInformation.xsd

XML Namespace
http://www.vocado.com/vm/fas
HTTP Endpoint
/channels/out.ev/consume
Message Exchange Pattern

Outbound event message (from SFP to client).

SFP publishes an outbound message that can be consumed by a client system. Please see MPG documentation for additional details on how messages are consumed.

Trigger Action

Message is triggered upon the following events met in the system (per award year):

  • ISIR codes, discrepancies and/or Verification requires clearance and all required document rules have been evaluated
    • ISIR codes include: ISIR Comment/ C-code, Reject Code, ISIR Assumption, or ISIR Highlight
  • Required document for the Valid ISIR process is received and is accepted or rejected
  • Additional supporting documents are requested within the Valid ISIR process
  • ISIR codes, Verification flag has been auto cleared, or discrepancies auto resolved/corrected
  • Manual ISIR Review task (i.e. code clearing) has been completed
  • Manual ISIR Verification task has been completed
  • ISIR status == Valid (process has been completed)
