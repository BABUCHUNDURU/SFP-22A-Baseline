Oracle SFP Messaging API 22.1.0.0

FasDocumentProcessStatusInformationRequest (Request/Reply)

webservicedocumentsync
Request Message Class
FasDocumentProcessStatusInformationRequest
Reply Message Class
FasDocumentProcessStatusInformationReply
Description

Service that returns information about documents per process for a given student.

XML Schema

FasDocumentProcessStatusInformation.xsd

XML Namespace
http://www.vocado.com/vm/fas
HTTP Endpoint
/channels/in.sync/publish-consume
Message Exchange Pattern

Synchronous inbound request-reply

A client system sends a message in the HTTP request and waits until the reply message is returned in the HTTP response. Please see "MPG Public API for Developers" for additional details.

By design, this mode of communication always operates in read-only mode. This means that it can only be used to retrieve data that is already available in SFP.

Request Details

It contains the following fields:

  • External Student ID (required)
  • Process Instance ID (optional)
  • Process Status (optional)
  • Document Status (optional)
  • StudentRequestFlag (optional)
Reply Details

It contains a list of tags according to rules below:

  • If Process Instance ID is provided, then only the process with the given ID will be returned (if it exists)
  • If Process Status is provided in the request, then only the processes in the given status will be returned
  • If Process Status is not provided in the request, then all processes will be returned
  • If Document Status is provided in the request, then only the documents in the given status will be returned (associated to their specific process)
  • If Document Status is not provided in the request, then all documents will be returned (associated to their specific process)
  • If the StudentRequestFlag is included in the request and set to true, documents that are configured to be hidden from the self-service UI will be excluded in the response.
Error Handling

If the request message contains validation errors, a MessageValidationErrorReply message will be returned in the HTTP response. Please check the payload of this message for details about the error.
