Oracle SFP Messaging API 22.1.0.0

FasSafiSyncEvent (Inbound)

programacademicdatasafiinbound
Message Class
FasSafiSyncEvent
Description

Inbound message that contains a student record sent by an external SRS (Student Record System) to SFP in order to synchronize student academic and financial information.

XML Schema

FasSafiSync.xsd

XML Namespace
http://www.vocado.com/vm/fas
HTTP Endpoint
/channels/in.ev/publish
Message Exchange Pattern

Inbound event message (from client to SFP).

When SFP receives a message, a Receipt ID is returned to the caller in the HTTP response. The actual message processing happens in background. For this reason, clients should monitor outbound messages for feedback.

Expected Outcome
  • SFP synchronises its version of the student's academic and financial information to match the SRS.
  • In a sequence of SAFI Synchronisation messages:
    • Information that appears in addition to information contained in a preceding message will be added to the FAS version of the student's academic and financial information
    • Information that no longer exists in a subsequent message will be deleted from the FAS version of the student's academic and financial information
    • Information that has been updated in a subsequent message will updated the FAS version of the student's academic and financial information
Error Handling

If the message contains validation errors, a MessageValidationErrorEvent message will be published by SFP. Please check the payload of this message for details about the error.
