Student Financial Planning : Product Reports

Created by Stephen Hutchinson, last modified by Fernando Medina on Dec 15, 2021

The following SFP Product Reports are deployed in the SFP Oracle BI Publisher instances.

Report AreaReport NameReport Description
ExecutiveMoney ReportThe Money Report is a report that shows the aging of students and/or disbursements that have not met specific criteria needed to disburse over time.
ExecutiveUser Statistics ReportThis report displays a count of tasks completed, released QC Holds, and any manually modified credit balances for a given time range by user at the summary and detail levels.
MonitoringApplication Event Error ReportProvides all application event errors (i.e. Integration errors) generated from SFP. Customers can use this report to diagnose issues when a student is not progressing as expected through their financial aid process.
MonitoringBatch Job Execution ReportShows the status of Batch Jobs within the specified hour interval. Possible statuses are Started, completed, abandoned (job exceeded duration threshold), or failed.
MonitoringBatch Job Schedule ReportPulls the environment's current Batch Job Schedule.
MonitoringBatch Job Statistics ReportThis report shows processing statistics for batch jobs in the financial aid application. It can be run on a specified date range and/or batch job.
MonitoringBatch Job Status ReportProvides a summary and detail list of batch job executions that failed.
MonitoringDelayed COD Responses ReportLists the COD message responses that have not been received for an Origination or Disbursement file that was sent to COD within the specified Start and End dates.
MonitoringOutstanding Financial Aid History Files ReportLists students without recent Financial Aid History (FAH) files within the specified Number Of Days Since NSLDS File Request.
MonitoringPackaging Triggers ReportShows the activities that triggered packaging for the specified time period.
MonitoringStudents Missing Date of Birth ReportLists active students whose Date of Birth is missing, which causes delays in triggering COD files and packaging for Title IV funds.
MonitoringSystem Release QC Hold Metrics ReportDisplays the percentage of QC holds that were system released within a specified time period. System released QC holds are holds released using the FasReleaseQcHoldEvent.
OperationalAward Letter Reconciliation ReportProvide information on awarded funds to allow for timely letter processing.
OperationalAwarding Criteria Status ReportProvides a list of students who have not met one of the award criteria for an academic year. Customers can use this report to identify why a student has not been awarded in an Academic Year.
OperationalAwarding Projected ReportSummary and Detail level of students with projected funds. Customers can use this report to proactively identify students who have been delayed in the awarding phase of the financial aid process.
OperationalAwarding Status ReportReport will provide an overview of the Awarding criteria as well as detailed tabs that will house specific student information in order to determine and actions that need to be taken by the College or University.
OperationalCommonline Discrepancy Handling ReportProvides a user predefined discrepancy reasons for Commonline loan that may require manual action to fix by a user
OperationalCommonline Error Handling ReportProvides a user with detailed information utilizing the Application/Loan Phase Code received in the Application Response file, file identifier code of R004P
OperationalCommonline Important Data ReportProvides a user important data elements for Commonline loans. Is linked to the Commonline Discrepancy Handling report to allow a user to understand if manual action is required on a CL loan
OperationalCredit Balance ReportListing of all students with pending Credit Balances. Customers can use this report to identify and resolve any outstanding pending credit balances for students.
OperationalDelayed Package ReportProvides students that have a delay in packaging that may require a manual repackage.
OperationalDisbursed Funds Without Credit Balances ReportReturns a list of students who have disbursed funds but do not have a credit balance after 3 days. Customers can use this report to verify if the financial aid system correctly triggered the credit balance calculation for students with disbursements.
OperationalDisbursement Criteria Failure ReportListing of disbursements that have failed a particular disbursement criteria. Customers can use this report to proactively identify actions required in order for a student's' upcoming disbursement to be disbursed by disbursement date.
OperationalDocument Information ReportProvides metrics of received documents by status and allows user to view individual student data. Customers can use this report to identify students with docs that may be in a status that blocks downstream processing (i.e. Doc Status = Requested).
OperationalDuplicate SSNs ReportLists students with duplicate SSNs, which causes data discrepancies across the application.
OperationalEnrollment Data Discrepancy ReportLists students who have a discrepancy between their internal enrollment IDs (prior enrollment ID = current enrollment ID) that causes issues across the student's enrollment including packaging and disbursing.
OperationalFA Cancellation Reconciliation ReportProvide information on when a cancellation letter creates and detailed cancel information.
OperationalFAS Process ReportSummary and detail of financial aid processes broken out by time frames. Customers can use this report to review summary and detail level data related to financial aid processes in SFP.
OperationalFAS Task ReportProvides metrics on task queues. Customers can use this report to highlight productivity within task queues.
OperationalFSEOG Allocation ReportDisplays summary and detai FSEOG Allocation information for an allocation schedule based on the Award Year Selected. Customers can use this report to validate FSEOG fund allocation data.
OperationalFund Acceptance ReportProvides an output for users to determine what status a disbursement is in as well as what their current fund acceptance status for that fund is.
OperationalGrant Overpayment ReportProvides a listing of Pending/Completed Grant Overpayments. Customers can use this report to check on the status of students and where they fall on the spectrum of repayment or being reported.
OperationalLetters History ReportSummary and detail of letters based on specific triggering events and statuses. Customer can use this report to view summary and student level data related to generated letters.
OperationalManual SAP Review ReportIdentifies all students whose SAP Manual Indicator is set to ""Yes"" within a specified date range. Customers can use this report to find students based on PP/Term Start and End Date that have a Manual SAP Indicator set to Yes.
OperationalMaterial Package Changes ReportProvide information on material changes in packages (Date changes and award amounts).
OperationalMPG API Errors ReportThis report returns errors from the message processing gateway for any non-200 response error code.
OperationalNFF Allocation ReportDisplays Allocation information for NFF funds based on the Award Year Selected. Customers can use this report to validate FSEOG fund allocation data.
OperationalNSLDS Record By Packaging Version ReportReport will provide detailed information from the NSLDS file that was used in determining a students eligibility at the time of a historical package.
OperationalQC Hold Analysis ReportReport displays total number of QC holds created between start and end dates that remain on hold, and displays total number of QC holds released between start and end regardless of when the hold was created.
OperationalQC Hold ReportList of quality control hold information at a summary/student level per hold type. Customers can use this report to download a .csv file to upload to a ticket management software (e.g. JIRA, WIP, etc. )
OperationalR2T4 CARES ACT ReportProvide a list of withdrawn students potentially affected by the R2T4 CARES Act guidelines
OperationalR2T4 ReportSummary and student level report of completed and pending R2T4s, including the associated pending reason. Customers can use this report to proactively identify populations of students with pending R2T4 calculations.
OperationalReturn Notification Reconciliation ReportProvides users the ability to compare letter creation dates and the DOD.
OperationalSAP Evaluations ReportSummary and student level details of completed SAP evaluations per program. Customers can use this report to proactively identify populations of students with a SAP status that may disqualify them from financial aid.
OperationalStudent Academic Year Override ReportList by student within a specified number of days to next disbursement that a user manually re-mapped an Academic Year. Customers can use this report to make sure newer SAFIs are accepted and mapped to ensure the most accurate disbursement.
OperationalStudent Groups ReportGenerate this report to help you identify which cohort of students has been added to or removed from a student group as of a given effective date.
OperationalStudent Milestone ReportListing of new students as they progress through the financial aid system.
OperationalStudent Record Hold ReportReturns student records within a specific Student Record Hold within a given date range.
RegulatoryFISAP Part II Other Disbursements ReportStudent disbursement level report for FISAP - Section 2 Pell and state funds sections, categorized per FISAP Total Income category (FTI). Customers can use this report to validate data against the aggregate values populated in the final FISAP report.
RegulatoryFISAP Part II ReportSummary and detail level report for FISAP - Section 2 dependent students section, categorized per FISAP Total Income category (FTI). Customers can use this report to validate data against the aggregate values populated in the final FISAP report.
RegulatoryFISAP Part VI ReportSummary and detail level report for FISAP - Section 2 independent students section, categorized per FISAP Total Income category (FTI). Customers can use this report to validate data against the aggregate values populated in the final FISAP report.
RegulatoryGainful Employment Report - Data InputsProvides FAS disbursement student data per program. Customers can use this report to gather necessary details to populate in the GE report template to submit to ED.
RegulatoryIPEDS Report - Data InputsProvides student level fund data. Customers can use this report to gather necessary details to populate in the Student Financial Aid (SFA) survey Integrated Postsecondary Education Data System (IPEDS).report template to submit to ED.
RegulatoryVerification Results ReportIdentity Verification Results for students flagged for ISIR verification in groups V4/V5. Customers can use this report to validate the data necessary to submit in the Verification Results Report required to submit to ED.
SAIG ManagementBlocked COD Transmission Reasons ReportReport of valid reasons preventing individual COD transmissions so they can be alleviated by the customer.
SAIG ManagementCOD Exceptions ReportDisplays COD files that did not load in the UI (accepted and rejected). Customers can gather all transactions that were transmitted to COD at a Document ID level and compare the corresponding response data from COD using the matching Document ID.
SAIG ManagementCOD Import Failure ReportIdentifies student funds that were not updated correctly when a COD file was received.
SAIG ManagementDaily COD Reject ReportSummary and detail of COD rejects received by day. Customers can use this report to review a summary of awards/disbursements that have been rejected by COD (Data is organized by reject code)
SAIG ManagementImported COD Transmissions ReportReturns records that have been transmitted and received from COD (both accepted and rejected). Customers can gather all COD transactions at a Document ID level and compare the corresponding response data from COD using the matching Document ID.
SAIG ManagementISIR Loading Integration ReportReport of the status and any issues related to the loading of ISIR files
SAIG ManagementISIR Reconciliation ReportProvides summary and detailed ISIR information. Users will be able to review any potential issues with ISIR loading or corrections as well as being able to find ISIRs that may need manual intervention to match.
SAIG ManagementOutstanding COD Reject ReportThis report populates all COD rejects outstanding at any given time. Customers can use this report to review awards/disbursements that have been rejected by COD, organized by rejects that have not yet been resolved.
SAIG ManagementSAIG Error COD ReportSummary and detail SAIG error information. Customers can use this report to review and correct awards/disbursements that have been rejected by COD, organized by date range.
SAIG ManagementSAIG Error CPS ReportSummary and detail SAIG error information. Customers can use this report to review and correct any errors received from CPS.
SAIG ManagementSAIG Error NSLDS ReportSummary and detail SAIG error information. Customers can use this report to review any NSLDS errors received from the error/acknowledgement file.
SAIG ManagementSAIG Inbound Errors ReportIdentify file importing/system level errors when receiving files from COD. Customers can use this report to identify/fix any inbound issues when receiving COD files.
SAIG ManagementSAIG Mailbox ReportSAIG mailbox activity. Customers can use this report to review and validate SAIG mailbox activity, i.e. files successfully transmitted / received from SFP to ED.
AuditAudit FAS User Role ManagementProvides a list of Audit FAS User Role Management logs.
AuditAudit LogProvides a list of Audit logs.
AuditAudit Login Logout AttemptsProvides a list of Audit Login Logout Attempts logs.
AuditAudit Portal User Role ManagementProvides a list of Audit Portal User Role Management logs.
AuditAudit Regulatory ManagementProvides a list of Audit Regulatory Management logs.
