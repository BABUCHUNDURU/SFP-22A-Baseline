Student Financial Planning : Task Examples

Created by Stephen Hutchinson, last modified on Oct 26, 2020

These are examples of queries that reference views in the Tasks subject.

The code convention $P{INPUT_PARAMETER} is single value input parameter and $X{IN, COLUMN_NAME, INPUT_PARAMETER} is the equivalent of a SQL Language "IN ()" statement. Both of these conventions are used specifically in Jasper Server and map to the input criteria defined within a report definition.

FAS Task Report
SELECT  c.queue_name
      , CASE WHEN a.total_created IS NULL THEN 0 ELSE a.total_created END     as  total_created
      , CASE WHEN b.total_completed IS NULL THEN 0 ELSE b.total_completed END as total_completed
      , b.average_completion_time
      , CASE WHEN b.average_completion_days IS NULL THEN 'N/A' ELSE b.average_completion_days END  as  average_completion_days 
FROM  (SELECT  queue_name   
       FROM FASRPT.TASK_QUEUE_DEFINITION_VW
       GROUP BY queue_name
      )    c
      LEFT JOIN (SELECT task_queue
                      , count(1)   as total_created
                 FROM FASRPT.TASK_VW    
                 GROUP BY task_queue
                )   a ON c.queue_name = a.task_queue
      LEFT JOIN (SELECT task_queue
                      , ((CAST(updated_datetime AS DATE) - CAST(created_datetime AS DATE)) * 24 * 60 * 60)   as average_completion_time 
                      , TO_CHAR(ROUND(AVG(CAST(updated_datetime as date) - CAST(created_datetime as date)), 0))  average_completion_days
                      , count(1)   as total_completed
                 FROM FASRPT.TASK_VW
                 WHERE task_status IN ('COMPLETION','CANCELLED')
                 AND   created_datetime BETWEEN $P{Start_Date} AND $P{End_Date}    
                 GROUP BY  task_queue
                         , ((CAST(updated_datetime AS DATE) - CAST(created_datetime AS DATE)) * 24 * 60 * 60)  
                )   b ON c.queue_name = b.task_queue
