Student Financial Planning : System Examples

Created by Stephen Hutchinson, last modified on Oct 26, 2020

These are examples of queries that reference views in the System subject.

The code convention $P{INPUT_PARAMETER} is single value input parameter and $X{IN, COLUMN_NAME, INPUT_PARAMETER} is the equivalent of a SQL Language "IN ()" statement. Both of these conventions are used specifically in Jasper Server and map to the input criteria defined within a report definition.

Batch Job Status Detail Report
SELECT a.job_name
     , a.job_execution_id
     , a.job_start_datetime
     , a.job_end_datetime
     , b.exception_class
     , b.exception_message
     , b.item_time
     , b.skip_location
     , b.step_name
FROM  FASRPT.BATCH_JOB_EXEC_HIST_VW    a
      LEFT JOIN FASRPT.BATCH_JOB_EXEC_HIST_ENTRY_VW b ON a.job_execution_id = b.job_execution_id
WHERE job_start_datetime BETWEEN $P{Start_Date} AND $P{End_Date}
AND $X{IN, job_name, Job_Name}
AND a.job_execution_id = NVL($P{Job_Execution_ID}, a.job_execution_id)
