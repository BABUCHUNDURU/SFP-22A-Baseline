POST /vug-api/admin/tdclient/password

Changes the credentials VUG uses to communicate with USDE. The client is responsible for effectively changing the mailbox password (USDE website) before invoking this service.

Since 118.11.0.0

Parameters

NameRequiredDescription
useryesThe username (TG Number)
passyesThe password (plain text)

Response

  • Header Content-Type: application/json
  • HTTP Status: 200
  • Payload
    • in case of success:  {"success": true}
    • otherwise: {"success": false}

HTTP 4xx or 5xx may also be returned in case of communication or validation errors.

Examples

Client successfully updates credentials
$ curl -d user=TG0000 -d pass=SECRET https://server/vug-api/admin/tdclient/passwd
{"success": true}


POST /vug-api/admin/tdclient/check

Checks if TDClient can connect to USDE using the stored credentials.

Since 118.11.0.0

No parameters

Response

  • Header Content-Type: application/json
  • HTTP Status: 200
  • Payload
    • if connection successful:  {"success": true}
    • if connection fails:{"success": false}

HTTP 4xx or 5xx may also be returned in case of communication or validation errors.

Example

Client confirms that VUG can connect to USDE
$ curl -X POST https://server/vug-api/admin/tdclient/check
{"success": true}

GET /vug-api/admin/tdclient/status

Retrieves TDClient status

No parameters

Response

  • Header Content-Type: application/json
  • HTTP Status: 200
  • Payload
    • if TD Client can communicate with SAIG Mailbox:  {"active": true}
    • if TD Client cannot communicate with SAIG Mailbox:{"active": false}

HTTP 4xx or 5xx may also be returned in case of communication or validation errors.

Example

Client resumes VUG operation
$ curl https://server/vug-api/admin/tdclient/status
{"active": true}

GET /vug-api/admin/info/ping

Checks if VUG is up. This endpoint can be invoked using the HTTP HEAD verb.
In both cases, the response will have a header with the current vug-api version (vocado-vug-api-version).

GET / vug-api/admin/export/files

Exports all files received in SAIG mailbox within a provided time range in zip format.  

Since 5.3.0.0

Parameters

ParamRequiredDescritpionDefault Value
starttimeyesstart timestamp in ISO8601 format (inclusive)-
endtimenoend timestamp in ISO8601 format (inclusive)now


Additional validation enforced on all requests:

    • max number of concurrent export operations: 2
    • max number of days in the timestamp interval: 31

Response

  • HTTP Status: 200
  • Header Content-Type: application/zip
  • Payload: zip of all files within the provided time range

Client exports saig files to a zip file
