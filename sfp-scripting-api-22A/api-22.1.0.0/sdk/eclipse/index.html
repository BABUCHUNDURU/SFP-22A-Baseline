How to use Eclipse

Note:

  • This page assumes that a new project will be created from scratch.
  • If you already have scripts you can just move the files to the proper place later.

Before you start

  1. Install Java JDK (version 8 or newer)
  2. Install Eclipse IDE
  3. Download the SFP Scripting API documentation for the target SFP version

Configure Groovy Support in Eclipse

  1. Check your Eclipse version.

  2. Go to the Groovy Eclipse Wiki, Releases section, and find the respective Release Update Site URL that matches your Eclipse version. Each Eclipse version will require a different update site. Copy this value.

  3. In Eclipse, open Help > Install New Software

  4. Click Add

  5. Set a name and paste the Release Update Site URL you copied in Step 2

  6. Expand Main Package and select the Eclipse Groovy Development Tools feature

  7. Click Next and follow the remaining prompts to complete the setup

Create empty project in Eclipse

  1. Open File > New > Project...

  2. Select Groovy Project

  3. Click Next

  4. Set Project Name

  5. Ensure that a valid JRE is selected.

  6. Click Finish

Configure SDK Jar as a project library

  1. Right-click on the project and select Build Path > Configure Build Path

  2. Select Libraries > Classpath > Add External JARs...

  3. Select file sfp-scripting-api-sdk-autocomplete-22.1.0.0.jar. This jar in included in the provided SFP Scripting API .zip file.

  4. Click Apply and Close to Confirm

Enable code completion for a given configuration script

Notes:

  • This process should be done only once for each script.
  • The package declaration is automatically ignored by SFP when evaluating configuration scripts.

Steps:

  1. Find documentation page
    • Each type of configuration script is documented by a separate .html page. For example, for nfrEligibility you should open this page.
  2. Update .groovy script
    • In the script page, copy the content indicated in "Snippet for code completion" and paste it at the top of your .groovy file. This snippet tells the IDE how to determine available symbols.
    • By default, the snippet assumes that your scripts will be created under a folder named src/scripts
      • src is the base folder for all scripts in the project.
      • If your script was created under another folder, then you should update the package declaration in the snippet to match your conventions. For example, if you create a script under a folder named src/nfr/baseline then your package should be nfr.baseline. This is not strictly required but it will avoid unnecessary warnings.
    • Replace the default package name CHANGE_ME by your actual package name.
